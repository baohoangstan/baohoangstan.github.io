# AGENTS.md

Personal blog + docs site built with **Docusaurus 3.7** (TypeScript config, React 18, MDX), deployed to GitHub Pages at https://baohoangstan.github.io.

## Commands

Use **Yarn** (there is a `yarn.lock`; CI runs `yarn install --frozen-lockfile`). Node >= 18.

- `yarn start` — local dev server (serves **one locale at a time**; default `en`)
- `yarn start --locale vi` — preview the Vietnamese site
- `yarn build` — production build. **This is the only verification gate** (see below)
- `yarn serve` — serve the built `build/` dir
- `yarn typecheck` — `tsc` (editor/type sanity only; not part of compilation)
- `yarn clear` — wipe `.docusaurus/` cache when the dev server behaves oddly
- `yarn write-translations --locale vi` — scaffold/refresh translatable UI strings

There is **no linter, formatter, or test suite**. Do not invent `yarn lint`/`yarn test`. Verify changes with `yarn build`.

## Build is strict — broken links fail it

`docusaurus.config.ts` sets `onBrokenLinks: "throw"`. Any broken internal link (including ones caused by moving/renaming a doc) **fails `yarn build`**. Always run `yarn build` after touching content, links, or routes.

## Content architecture (non-obvious)

There are **three independent content roots**, each a Docusaurus plugin:

| Dir | Plugin / route | Notes |
|---|---|---|
| `blog/` | blog preset → `/blog` | authors in `blog/authors.yml`, tags in `blog/tags.yml` |
| `docs/` | docs preset → `/docs` | sidebar `docSidebar` |
| `about/` | **second `@docusaurus/plugin-content-docs` instance** (id `about`) → `/about` | sidebar `aboutSidebar`. These are docs, NOT React pages |

`src/pages/` holds real React pages (e.g. the homepage `index.tsx`); `src/components/` and `src/css/custom.css` hold shared UI/theme.

## Sidebars are auto-generated — don't hand-edit

`sidebars.ts` uses `{ type: "autogenerated", dirName: "." }` for both `docSidebar` and `aboutSidebar`. To change navigation:
- Add/move Markdown files under `docs/` or `about/` — the sidebar updates automatically.
- Set a folder's display name with a `_category_.json` (`{ "label": "..." }`).
- Order items with `sidebar_position` frontmatter (see `docs/intro.md`, `about/goal.md`).

## i18n (en + vi)

Locales: `en` (default) + `vi`. Translated content lives under **plugin-namespaced** paths, not mirroring the source tree directly:
- Docs: `i18n/vi/docusaurus-plugin-content-docs/current/<same path as docs/>`
- About: `i18n/vi/docusaurus-plugin-content-docs-about/current/...`
- Blog: `i18n/vi/docusaurus-plugin-content-blog/...`

Translation is **partial**; untranslated pages fall back to English automatically. When adding a translation, copy the file to the matching `i18n/vi/...` path.

## Deployment

Pushing to **`main`** triggers `.github/workflows/deploy.yml`, which runs `yarn build` and publishes `build/` to GitHub Pages via the Pages artifact. PRs to `main` run `test-deploy.yml` (build only). 

Note: `deploymentBranch: "deploy"` and `yarn deploy` (`docusaurus deploy`) in config are the legacy manual push-to-branch method and are **not** what CI uses — prefer the `main`-push flow.

## Gotchas

- `.gitignore` excludes `/build`, `.docusaurus`, and AI dirs (`.claude`, `.opencode`, `.omo`).
- `trailingSlash: false` — keep internal links without trailing slashes.
- A Google Analytics `gtag` ID is hardcoded in `docusaurus.config.ts`.
